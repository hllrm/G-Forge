### #21 fix spec (scoped 2026-07-13) — argv-parse commit-detection

⬜ spec only, not started. Executable by a fresh session / the ADR-004 pass. **Do not implement as an independent patch** — this defines the shared detection routine ADR-004's native `pre-commit` hook must reuse.

**Root cause — two string-op sites in `hooks/check-commit.sh`:**
- **Surface #1 (detection, `:66`):** matches a `git … commit` substring by command *content*, not by which command actually runs → fires on non-committing commands that merely contain `git commit ` followed by whitespace/args. Reproduced live on current `main` this session: a Bash `for`-loop whose quoted arg held `git commit -m z` was denied though it ran no git. (`f55ccdb` closed only the trailing-quote sub-case — `…git commit"` no longer fires — but `git commit <space>…` inside any string still does. The original `grep -iE "git add|git commit" …` from the BUG-2 report reflects a **pre-`f55ccdb` stale installed hook** in G-Tools, not current `main`.)
- **Surface #3 (pathspec extraction, `:127`):** `sed -E 's/^.*[[:space:]]commit([[:space:]]|$)//'` greedily slices after the **last** `commit` token and `xargs`-tokenizes the rest → a `-m` message body citing real paths is parsed as pathspecs → misclassified **mixed**.
- **Surface #2 (chained pre-exec state):** inherent to PreToolUse — `… && git add x && git commit` in one call is evaluated pre-`git add`. **Not fixable here; stays for ADR-004's `pre-commit` hook (index authoritative).**

**Fix — one sourced routine (`hooks/lib/commit-detect.sh`) reused by PreToolUse + ADR-004's `pre-commit` (no second regex):**
1. Identify the actual command: strip leading `VAR=val`/`env`; require the first real token to be `git` (bare or `…/git`). Not git → not a commit → **pass** (kills #1).
2. Confirm subcommand: walk argv skipping value-taking global flags (`-C <path>`, `-c <k=v>`); first non-flag token must be exactly `commit`. A quoted `commit` substring is never the subcommand token.
3. Pathspecs from argv only: skip option-values (`-m <msg>`, `--message=…`, `-F <file>`, `--author=…`); treat only bare positionals (after `--`/non-flag) as pathspecs — the `-m` value is never a pathspec (kills #3).
4. No whole-string `sed`-slice. If argv parsing is defeated (nested quotes/subshell), **fail safe toward gating** (stricter code bucket) — but never fire when the first command isn't git.

**Test plan (extend `tests/test-check-commit.sh`, fail-before/pass-after):**
- T-a #1 (regression guard — already passes post-`f55ccdb`): `grep -iE "git add|git commit" retros/*.md` (ends `commit"`) → stays NOT blocked. (The true fail-before for #1 is T-b, which fires on current `main`.)
- T-b #1/#3 (true fail-before on current `main`): `echo`/`for` whose quoted arg holds `git commit -m z` → currently **fires**; must NOT block after fix.
- T-c #3: `git commit -m "touches hooks/x.sh and g-docs/y.md"`, doc-only staged set + doc sentinel → classified **doc**, not mixed.
- T-d (no #7 regression): `git commit -- hooks/x.sh` real pathspec + code staged → still **code**.
- T-e (no #6 regression): `git -C /repo commit`, `git -c k=v commit` → still detected.
- T-f (core gate): bare `git commit`, code staged, no sentinel → still **blocked**.
- Existing 22-test suite stays green.

**Constraints:** shared routine mandatory; does **not** close BUG-2 fully (#2 waits for ADR-004 — never record #21 resolved until then); atomic `/g-update` rollout with #5 drift check flagging a partial hook-set update; gate self-modification → mandatory `/g-review` + full suite green with pasted output (finding #20 / g-rules-H). **Estimate:** ~1 wave (implementer + test-writer).
